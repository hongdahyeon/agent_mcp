# 기술 스택 결정 및 문제 해결 분석 (블로그용)

`tasks.md` 및 `implementation_plan.md`의 프로젝트 이력을 바탕으로, "기술 스택 및 의사 결정" 블로그 포스팅에 적합한 주요 기술적 도전 과제와 결정 사항들을 정리했습니다.

## 기술적 결정 및 문제 해결 요약표

| 카테고리 | 문제 상황 / 결정 사항 | 기술적 접근 방식 | 결과 및 이점 |
| :--- | :--- | :--- | :--- |
| **프론트엔드 아키텍처** | **React & TypeScript 마이그레이션** | Vite + React + TS 도입 (기존 HTML/JS에서 전환) | 유지보수성 및 타입 안정성 향상. 복잡한 SSE 상태 관리(데드락, 클로저 문제) 해결. |
| **실시간 데이터** | **실시간 도구/로그 스트리밍** | **SSE (Server-Sent Events)** 적용 | WebSocket보다 가벼운 단방향 통신으로 실시간 대시보드 및 터미널 스타일 로그 구현. |
| **보안 (Password)** | **비밀번호 보안 표준화** | **Bcrypt (passlib)** 도입 (기존 SHA256에서 전환) | 레인보우 테이블 공격 방어 강화. 라이브러리 버전 고정으로 호환성 이슈 해결. |
| **보안 (Auth)** | **세션 및 API 인증 통합** | **JWT (JSON Web Token)** 도입 | 웹 세션(12시간)과 장기 API 키(1년) 인증 체계를 표준화된 헤더 방식으로 단일화. |
| **확장성** | **동적 도구(Tool) 확장성** | **Pydantic 동적 모델 생성** | 서버 재시작 없이 런타임에 SQL/Python 도구를 생성하고 즉시 사용할 수 있는 환경 구축. |
| **백그라운드 작업** | **예약 메일 발송 기능** | **APScheduler** 활용 | 메일 발송 로직을 요청-응답 주기와 분리하여 백그라운드에서 안정적으로 예약 처리. |
| **계층 분리** | **거대 DB 매니저 리팩토링** | **모듈형 패키지 구조화 (`src/db`)** | 1000줄 이상의 `db_manager.py`를 기능별(user, log 등)로 분리하여 가독성 및 테스트 용이성 확보. |
| **상호 운용성** | **레거시 XML API 대응** | **OpenAPI Proxy & `xmltodict`** | 공공데이터 규격(XML)을 프런트엔드에서 다루기 쉬운 JSON으로 자동 변환하는 프록시 계층 구축. |
| **인프라** | **Stdio/SSE 통합 백엔드** | **인증 브릿지 계층 구현** | 동일한 비즈니스 로직을 Claude Desktop(Stdio)과 웹 UI(SSE)에서 공통으로 사용하도록 표준화. |
| **디자인/UX** | **시각적 완성도 향상** | **Glassmorphism & Tailwind CSS** | 반응형 사이드바와 블러 효과를 적용해 전문적이고 현대적인 대시보드 UI 구현. |

## 추천 블로그 포스팅 주제

1. **"Vanilla JS에서 React로: 실시간 MCP 대시보드 전환기"**
   - 내용: 복잡한 SSE 상태 처리, Hook에서의 Stale Closure 해결 방법, TypeScript를 이용한 도구 스키마 정의의 이점.
2. **"Python Bcrypt 호환성 늪에서 탈출하기: 트러블슈팅 사례"**
   - 내용: `passlib`와 `bcrypt` 최신 버전 간의 충돌 원인 분석과 종속성 피닝(Pinning)을 통한 서비스 복구 과정.
3. **"런타임에 코드를 실행하는 법: MCP 동적 도구 실행기 설계"**
   - 내용: Pydantic의 `create_model`을 활용한 동적 스키마 생성과 안전한 Python/SQL 실행 환경 구축 노하우.
4. **"Stdio와 SSE를 넘나드는 멀티 프로토콜 에이전트 서버 아키텍처"**
   - 내용: 로컬 CLI용(Claude Desktop)과 원격 웹용 로직을 하나로 통합하여 관리하는 백엔드 설계 전략.
